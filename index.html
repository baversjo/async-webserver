Introduction

This site presents our project in the programming course Networkprogramming (EDA095) which is a very fast, highly performant and asynchronous HTTP file server. Our server is written in Java (Java 1.7) and uses the native sendfile(2) linux call.

Manual

Because the project is a webserver the user manual will be adapted to the knowledge of an network admin.

The first thing to do is to create a directory where all the html documents will be stored. You do this because the server needs to have all the files on a mutual directory to hace acces to them.

The program which will be resposible for the processes needs to hace access to the directory where the files are contained. To get access to the directory we have created a variable which is an environment variable. This variable gives the program the correct searchpath for the directory. The admin knows how to set such a variable for the searchpath. Something else the admin should consider to fix before the server works is to the a specific port number. We have also created an enviroment variable for this to make it easy for the admin.

We used the library from Java 1.7 to create the server. This result in so the admin needs to install Java 1.7 or a newer version. If you use an older version it will not work. This beacuse Java programs are not compatible with older version of Java.

The last step is to execute the main program "Server.java" in a suitable compilor, i.e Eclipse.
